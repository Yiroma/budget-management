# Conventions du projet Budget Management

Ce dossier regroupe l'ensemble des conventions adoptées pour le projet. Chaque fichier couvre un domaine spécifique.

## Sommaire

1. [Git](./01-git.md) — Branches, commits, pull requests, versioning
2. [Code Style](./02-code-style.md) — Formatage, nommage, outils (Prettier, ESLint, EditorConfig)
3. [Frontend](./03-frontend.md) — React, Next.js, TypeScript, structure des dossiers
4. [Backend](./04-backend.md) — Java, Spring Boot, architecture en couches
5. [API](./05-api.md) — Conventions REST, status codes, format de réponse
6. [Base de données](./06-database.md) — Nommage, migrations, index
7. [Tests](./07-testing.md) — Stratégie, nommage, patterns
8. [Qualité](./08-quality.md) — Accessibilité, sécurité, performance, principes (DRY, SOLID, KISS, YAGNI)

## Règle générale

Toute contribution au projet doit respecter ces conventions. En cas de doute, se référer au fichier correspondant. Si une convention manque ou nécessite une mise à jour, ouvrir une PR dédiée.
